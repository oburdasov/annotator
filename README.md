В основе логики зума лежит transform: scale().
Можно было также сделать его через typescript, вручную устанавливая размеры элементов, и позицию аннотаций.
Сам зум, в таком случае, был бы менее производительным.

Но из-за transform усложняется логика перемещения аннотаций, т.к. всегда нужно учитывать коэффициент зума.
Честно говоря, мне сложно судить, какой вариант лучше в плане простоты кода и производительности. Буду рад услышать это в фидбэке.

Задание постарался сделать быстро, поэтому оставил некоторые баги/недочёты (надеюсь, не грубые):

1. При зуме кнопки для удаления и перемещения тоже меняют размер
2. Аннотацию можно создать слегка за границей, при перемещении она прыгнет внутрь документа
3. После зума, перемещение аннотаций за рамки немного нарушено
4. Если скроллить и перемещать аннотацию одновременно, выглядит скачкообразно
5. В мобайле вроде не работает перемещение. По крайней мере, в developer tools

Насчёт кода тоже есть некоторые сомнения, хоть и перепроверил всё несколько раз, но глаз замылился. Надеюсь, не упустил что-то критическое
